Admin the FLOW3 Administration Package

Introduction

Inspired by the awesome Django Admin this package tries to port the principles to this great new world that is FLOW3

Everybody who has already worked with the Django Admin knows how much tedious work it gets off your back right from the start. Reeinventing proper userinterfaces and interactions from the start everytime doesn't make sense in most cases.

Thanks to some of the amazing features in FLOW3, namely the Domain-Driven-Design, Annotations, Aspects and some other features it is finally possible to create an awesome Admin Interface for the PHP world.

Installation

Adding to an existing FLOW3 Installation

  1. Download: Admin.zip
  2. Unpack it to your Packages/Applications directory
  3. Activate the Package through the flow3 command line interface:
    ./flow3 package:activate Admin

Screenshot Walkthrough

Login

  • Easy and intuitive
  • Generates default User on first launch
  • Errors are displayed through flashmassages

Dashboard

  • Topbar Navigation
  • Widgets
  • Sidebar Navigation
  • Uses the awesome Bootstrap Framework for the UI

Listview

  • Clean and easy
  • Intelligient Limits (depending on the amount of records)
  • Pagination
  • Sorting
  • Bulk Actions: By default there is an action to delete any number of records. Adding additional Actions is quick and easy.

Update

  • Clean and easy
  • Intelligient Limits (depending on the amount of records)
  • Pagination
  • Bulk Actions

Bulk Delete

  • Every action can have a seperate View
  • The Delete action shows an confirmation page to delete one or more items

Roles

  • Usermanagement is configured through roles
  • you can define any number of roles and assign them to a user
  • you can set policies to grant for roles
  • policies are automatically generated based on available models and actions (including your custom actions)

Validation

  • Validation is based on FLOW3 entity validation. You just need to define your validation in your entity and the Admin takes care of the rest!
  • Default actions include (required, e-mail, alphanumeric, number, stringlength, datetime, ...)
  • Custom Validations are easily added as well

Widgets

  • Widgets are automatically assigned by entity property type
  • Default Widgets include TextField, Textarea, Select, MultiSelect, Upload, RTE
  • Overwriting an widget for a specific model of package is as easy as creating a specific template
